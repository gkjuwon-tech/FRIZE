# 🔥 FRIZE

### Firefighting Real-time Integrated Zone-control Environment

> "불은 뜨겁지만, 지휘는 차가워야 한다."
> — FRIZE 사내 명언인데 사실 방금 지음

**작성일:** 2026-06-17

## 0. 들어가기 전에 (읽기 싫어도 읽어)

자 상상을 해보자.

지금 소방 현장은 이렇다. 무전기로 "야 거기 3층! 3층! 어 안 들려? 야!! 3층!!!" 하고 고래고래 소리지른다. 대원은 연기 속에서 손전등 하나 들고 "여기가 어디지" 하면서 벽 더듬는다. 지휘관은 건물 밖에서 도면 들고 "여기가 여긴가" 하면서 머리 굴린다. 정보는 사람 머릿속에 흩어져 있고, 명령은 목소리로 날아가다 매연 속에서 증발한다.

이게 2026년에 맞는 그림이냐? **아니지.**

FRIZE는 이 모든 난장판을 **하나의 운영체제**로 묶는다. 윈도우가 마우스·키보드·모니터·프린터를 추상화해서 "딸깍"으로 굴리듯이, FRIZE는 **드론·스마트고글·소방장비·대원·건물**을 추상화해서 지휘관이 콕핏에서 "딸깍"으로 굴린다.

PC에서 폴더 더블클릭하면 열리지? FRIZE에서는 콕핏에서 위험구역 딸깍하면 거기에 드론이 날아가고, 대원 딸깍하면 그 사람 고글에 명령이 뜬다. **그게 OS다.** 진짜 OS는 아닌데 개념적으로 OS다. 우리가 OS라고 부르면 OS다. 토 달지 마.

## 1. 이게 도대체 무슨 물건인가 (Elevator Pitch — 근데 그 엘리베이터가 불남)

**FRIZE는 소방 현장을 위한 "지휘 운영체제(Command OS)"다.**

- **입력장치:** 정찰 드론, 대원 스마트고글, 열화상 센서, 건물 IoT
- **출력장치:** 대원 고글의 AR 명령, 자동 소방장비, 드론 액추에이터
- **커널:** 현장을 3D로 통합·해석하는 상황인식 엔진
- **유저:** 콕핏에 앉은 지휘관 (그리고 현장의 대원들)
- **마우스 클릭:** "딸깍" (™ 출원 예정 농담임)

한 문장으로: **"현장을 스캔 → 콕핏에 띄움 → 딸깍 → 현실이 움직임."**

이게 끝이다. 근데 이 한 줄 뒤에 엔지니어 30명의 영혼이 갈려 들어간다.

---

## 2. 왜 만드는가 (Problem Statement — 진지하게 들어감, 5초만)

웃긴 톤이라 그렇지 이 문단은 진짜다:

1. **정보 단절** — 현장 안(대원)과 밖(지휘관)이 같은 그림을 못 본다. 대원은 자기 발끝밖에 못 보고, 지휘관은 건물 외벽밖에 못 본다.
2. **명령 손실** — 무전은 시끄럽고, 끊기고, 헷갈린다. "왼쪽"이 누구 기준 왼쪽이냐.
3. **반응 지연** — 위험을 인지하고 → 판단하고 → 전달하고 → 행동하기까지 단계마다 초가 깎인다. 소방에서 초는 곧 생명이다.
4. **장비 따로국밥** — 드론 조종사 따로, 펌프 담당 따로, 진입 대원 따로. 통합 지휘가 안 된다.

FRIZE는 이 4개를 **하나의 화면, 하나의 손가락(딸깍)** 으로 뭉갠다.

---

## 3. 핵심 컨셉: "현장이 곧 데스크탑이다"

PC 운영체제에 빗대보자. 이 표 하나로 전체 철학이 설명된다.

| 일반 OS | FRIZE | 설명 |
|---|---|---|
| 바탕화면 | **실시간 3D 현장 트윈** | 현장을 통째로 디지털로 복제한 화면 |
| 아이콘 | **대원·드론·장비·위험요소** | 화면 위에 떠다니는 클릭 가능한 객체들 |
| 마우스 클릭 | **콕핏 딸깍** | 객체 선택 → 명령 발동 |
| 프린터로 출력 | **대원 고글로 명령 출력** | "출력장치 = 사람 눈앞" |
| 웹캠 입력 | **고글 1인칭 시야 입력** | 대원 시야가 콕핏으로 역송출 |
| 백그라운드 프로세스 | **드론 자동 정찰** | 알아서 돌아다니며 스캔 |
| 알림(Notification) | **위험 경보 팝업** | "야 거기 무너진다" |
| 작업 관리자 | **대원 생체·산소·위치 모니터** | 누가 위험한지 한눈에 |

즉 FRIZE는 **"현실 세계를 클릭 가능한 인터페이스로 바꾸는 추상화 계층"** 이다. 멋있지? 방금 만든 말이다.

## 4. 시스템 구성 (Architecture — 그림은 없지만 글로 그려줌)

```
            ┌───────────────────────────────────────────────┐
            │                 콕핏 (Cockpit)                  │
            │   3D 현장 트윈 │ 대원 시야 멀티뷰 │ 딸깍 명령패널  │
            └───────────────┬───────────────────────────────┘
                            │  (명령 ↓ / 영상·데이터 ↑)
            ┌───────────────┴───────────────────────────────┐
            │              FRIZE 커널 (Core Engine)           │
            │  • 센서 퓨전(드론+고글+열화상+IoT 합치기)         │
            │  • 3D 현장 재구성 & 위험도 추론                  │
            │  • 명령 라우팅(누구에게/무엇을/어떻게)            │
            │  • 안전 인터록(말도 안 되는 명령 차단)           │
            └───┬───────────┬───────────┬───────────┬────────┘
                │           │           │           │
          ┌─────┴───┐ ┌─────┴────┐ ┌────┴─────┐ ┌───┴──────┐
          │ 정찰드론 │ │ 스마트고글 │ │ 소방장비  │ │ 건물 IoT │
          │ (눈+팔) │ │ (눈+귀+입)│ │ (펌프/문) │ │ (감지기) │
          └─────────┘ └──────────┘ └──────────┘ └──────────┘
```

### 4.1 계층 구조 (진짜 OS처럼 레이어 나눔)

- **L0 — 하드웨어 계층 (HAL, Hardware Abstraction Layer):** 드론·고글·펌프·밸브·센서. 제조사 제각각인 장비를 표준 드라이버로 추상화. "어느 회사 드론이든 FRIZE에선 똑같이 딸깍."
- **L1 — 센서 퓨전 계층:** 들어오는 모든 영상/열/위치/음성/가스 데이터를 하나의 좌표계로 합침. 여기가 제일 빡센 곳. 엔지니어 운다.
- **L2 — 월드 모델 계층:** 현장을 실시간 3D로 재구성. "디지털 트윈". 어디가 불, 어디가 사람, 어디가 무너질지.
- **L3 — 인지/판단 계층:** 위험도 추론, 우선순위 큐("이 사람부터 빼"), 경로 계산. **VLM(비전-언어 모델)이 현장 판단을 전담한다.** 룰 기반 디텍터로 일일이 if문 박는 거 아니다 — 모델이 보고 판단한다.
- **L4 — 명령/오케스트레이션 계층:** 콕핏 딸깍을 실제 장비 동작으로 번역. 안전 인터록 통과해야 발동.
- **L5 — 프레젠테이션 계층:** 콕핏 UI + 대원 고글 AR. 사람이 실제로 보는 부분.

> 📝 설계 메모: L3 판단을 VLM에 몰아주는 건 의도된 선택이다. 현장은 매번 다르고 룰로 다 못 잡는다. "사람이 쓰러져 있다 / 천장이 처졌다 / 저 연기 색이 이상하다" 같은 걸 모델이 보고 말로 설명하게 한다. 룰 디텍터는 안 쓴다.

## 5. 핵심 시나리오 (이거 읽으면 "아 이래서 만드는구나" 함)

### 시나리오 A — "딸깍 한 번에 사람을 살린다"

1. 🚒 출동. FRIZE 부팅. 콕핏 화면에 "현장 없음 → 스캔 중..." 로딩.
2. 🛸 **정찰 드론 3대 자동 출격.** 건물 외벽 한 바퀴 돌면서 열화상 스캔. 사람 머릿속이 아니라 화면에 3D 건물이 **솟아오름.**
3. 🧠 커널이 분석: "3층 동쪽, 38℃ 넘는 발열체 2개 — 사람일 확률 높음. 2층 천장 처짐 — 붕괴 위험 67%."
4. 👁 콕핏 화면 3층 동쪽이 **빨갛게 깜빡.** 지휘관이 거기 **딸깍.**
5. 📢 그 구역 담당 대원의 **스마트고글에 화살표 + "여기 사람 2명, 진입로 →" AR로 뜸.** 무전으로 "어 그 어 왼쪽 아니 내 왼쪽" 안 해도 됨.
6. 👀 동시에 대원 고글 1인칭 시야가 콕핏으로 **역송출.** 지휘관은 대원 눈으로 본다. "어 거기 문 닫혀있네, 부숴."
7. 🔨 지휘관이 그 문 객체 **딸깍 → "강제진입"** → 대원 고글에 "이 문 부수세요" 표시. (장비가 알아서 부수는 버전도 로드맵에 있음.)
8. 🚿 동시에 2층 붕괴 위험 구역에 **소방장비(자동 방수포) 딸깍 → 자동 조준·방수.** 사람 안 들어가도 됨.
9. ✅ 사람 2명 구조. 대원 전원 복귀. 콕핏에 "MISSION CLEAR" — 근데 이건 게임 아니니까 그냥 "상황 종료"로 뜸.

**걸린 시간: 무전 고래고래의 절반.** (수치는 우리가 만들 거니까 일단 이렇게 적자.)

### 시나리오 B — "대원이 위험한데 본인은 모름"

- 진입 대원의 산소 잔량 18%, 심박 178, 그리고 뒤쪽 복도 온도 급상승.
- 본인은 집중하느라 모름.
- FRIZE 커널이 먼저 캐치 → 콕핏에 **빨간 경보** + 해당 대원 고글에 **"30초 내 후퇴, 경로 ←" 강제 표시.**
- 지휘관이 후퇴 경로 **딸깍** → 대원 고글에 탈출 화살표가 바닥에 깔림(AR 내비). 연기 속에서도 선만 따라가면 됨.

### 시나리오 C — "드론이 사람 손보다 먼저 간다"

- 진입 너무 위험한 구역 → 사람 대신 **드론 딸깍 → 그 구역 정찰.**
- 드론이 보내온 영상에서 VLM이 "가스통 발견, 폭발 위험" 판단 → 전 대원 고글에 **공통 경고** 자동 푸시.

## 6. 콕핏 (지휘관 쪽) — 어떻게 생겼나

콕핏은 FRIZE의 "데스크탑"이다. 화면 구성:

- **🗺 메인 3D 트윈 뷰:** 현장 전체를 돌려보고 확대/축소. 층 단면 잘라보기. 불·연기·사람·위험요소가 색으로 구분.
- **👁 대원 멀티뷰 패널:** 모든 대원의 1인칭 고글 시야가 작은 창들로. 클릭하면 풀스크린. 넷플릭스 썸네일처럼 생김 근데 내용은 불바다.
- **🎛 딸깍 명령 패널:** 객체 선택하면 가능한 명령이 뜸. ("진입 / 후퇴 / 정찰 / 방수 / 강제진입 / 집결")
- **🚨 경보 스택:** 위험 알림이 쌓임. 심각도순 정렬.
- **📊 자원 현황 바:** 대원별 산소·심박·위치, 드론 배터리, 펌프 수압.
- **⏪ 타임라인 스크러버:** 상황을 되감아 볼 수 있음. "5분 전엔 불이 어디였지?" 딸깍하면 과거로.

> 인터랙션 원칙: **"세 번 딸깍 안에 모든 명령이 끝나야 한다."** 불나는 중에 메뉴 7단계 들어가게 만들면 우리가 불에 탄다.

## 7. 스마트고글 (대원 쪽) — 어떻게 생겼나

대원 고글은 FRIZE의 "출력장치이자 입력장치"다. 사람이 프린터이면서 웹캠인 셈. (사람한테 이렇게 말하면 화냄. 속으로만 생각하자.)

**출력(대원이 보는 것):**
- AR 명령 표시: 화살표, 하이라이트, "여기 사람 2명" 텍스트.
- 바닥 AR 내비게이션 라인 (연기 속 탈출용).
- 본인 생체신호 (산소·심박) 한 귀퉁이에.
- 동료 위치 (벽 너머도 점으로).

**입력(고글이 보내는 것):**
- 1인칭 영상 → 콕핏 역송출.
- 음성 ("3층 클리어") → 텍스트화되어 콕핏 로그에.
- 위치·자세·움직임.
- 헬멧 부착 열화상/가스 센서.

**철학:** 대원에게 정보를 쏟아붓지 않는다. **"지금 이 순간 필요한 한 가지"** 만 띄운다. 연기 속에서 대시보드 읽을 시간 없다. 화살표 하나, 그게 다다.

## 8. 기능 리스트 (Feature List — 너가 원한 거 여기 있다)

### 8.1 🛰 정찰 & 스캔 (Recon & Scan)

- **F-01 자동 드론 출격:** 출동 시 드론이 알아서 떠서 외곽 정찰.
- **F-02 실시간 3D 현장 재구성:** 드론+고글 영상으로 건물 디지털 트윈 생성.
- **F-03 열화상 발열체 탐지:** 사람/화점 후보를 화면에 표시.
- **F-04 위험요소 인식(VLM):** 가스통·균열·붕괴징후·역류 등을 모델이 보고 판단.
- **F-05 연기·시야 보정:** 연기 너머를 센서로 추정해 윤곽선으로 그려줌.
- **F-06 변화 감지:** "방금 전과 비교해 불이 번진 방향" 표시.

### 8.2 🧠 상황 인식 & 판단 (Cognition)

- **F-07 위험도 스코어링:** 구역별 위험 점수 실시간 갱신.
- **F-08 구조 우선순위 큐:** "누구부터 빼야 하는가" 자동 정렬.
- **F-09 붕괴/폭발 예측:** 모델 기반 사전 경고.
- **F-10 대원 안전 모니터:** 산소·심박·체온·고립 여부 추적.
- **F-11 경로 계산:** 진입로·탈출로 실시간 산출(불 번지면 재계산).

### 8.3 🎛 지휘 & 명령 (Command — "딸깍" 본진)

- **F-12 객체 딸깍 선택:** 화면 위 사람/드론/장비/구역 클릭.
- **F-13 컨텍스트 명령 메뉴:** 선택 객체에 가능한 명령만 노출.
- **F-14 대원 고글로 명령 푸시:** 딸깍 → 해당 대원 AR에 시각 명령.
- **F-15 구역 단위 일괄 명령:** "이 구역 전원 후퇴" 한 번에.
- **F-16 강제진입/방수/정찰 등 액션:** 장비·드론 직접 구동.
- **F-17 안전 인터록:** 위험한 명령(예: 붕괴 구역 진입)은 경고+2차 확인.

### 8.4 👁 시야 공유 (Vision Sharing)

- **F-18 대원 1인칭 역송출:** 모든 고글 시야를 콕핏에서.
- **F-19 멀티뷰 동시 모니터:** 여러 대원 동시에.
- **F-20 시야 위 주석(Annotation):** 지휘관이 대원 화면에 직접 동그라미/화살표.
- **F-21 드론 시야 전환:** 드론 카메라로 시점 점프.

### 8.5 🤖 자동화 & 장비 (Automation)

- **F-22 자동 방수포 조준:** 화점 딸깍 → 자동 조준·방수.
- **F-23 드론 자율 정찰 모드:** 지정 구역 알아서 순찰.
- **F-24 IoT 연동:** 건물 스프링클러·방화문·환기 제어.
- **F-25 장비 상태 모니터:** 수압·배터리·잔여 자원.

### 8.6 📡 통신 & 기록 (Comms & Log)

- **F-26 음성→텍스트 로그:** 모든 교신 자동 기록.
- **F-27 타임라인 리플레이:** 상황 되감기.
- **F-28 사후 보고서 자동 생성:** 출동 끝나면 리포트 자동 초안.
- **F-29 오프라인/저대역 모드:** 통신 끊겨도 핵심 기능 유지(현장은 망 안 좋음).

### 8.7 🛡 안전·신뢰 (Safety & Trust) — 농담 빼고 제일 중요

- **F-30 휴먼 인 더 루프:** 생명 직결 명령은 반드시 사람 확인.
- **F-31 페일세이프:** 시스템 다운 시 장비는 안전 상태로.
- **F-32 명령 감사 추적:** 누가 언제 뭘 딸깍했는지 전부 기록.
- **F-33 판단 근거 표시:** VLM이 "왜 위험하다 했는지" 설명을 같이 띄움(블랙박스 금지).

## 9. 만들면 안 되는 것 / 안 만들 것 (Non-Goals — 이거 안 적으면 나중에 싸움남)

- ❌ **무인 자동 소방** 아님. 사람을 대체하는 게 아니라 **사람을 증강**한다. 최종 판단은 사람.
- ❌ **장난감 드론쇼** 아님. 화려한 시각화가 목적이 아니라 생명이 목적.
- ❌ **만능 AI 신탁** 아님. 모델은 틀릴 수 있고, 그래서 근거를 보여주고 사람이 확인한다.
- ❌ **소방관 감시 도구** 아님. 생체 모니터는 안전용이지 근태관리용 아님. (이거 오해하면 현장에서 안 씀.)

## 10. 단계별 로드맵 (현실적으로, 근데 야망 있게)

**Phase 0 — "화면 위 장난감" (PoC)**
- 가짜 데이터로 콕핏 3D 트윈 + 딸깍 인터랙션만. 실제 장비 없음.
- 목표: "딸깍하면 명령 간다"의 손맛 검증.

**Phase 1 — "한 명과 한 대"**
- 고글 1대 + 드론 1대 + 콕핏. 실내 모의 현장.
- 시야 역송출 + 명령 푸시 왕복 검증.

**Phase 2 — "퓨전"**
- 다수 대원·다수 드론 + 센서 퓨전 + 3D 재구성.
- VLM 판단 계층 투입.

**Phase 3 — "장비가 움직인다"**
- 자동 방수포·IoT 연동. 안전 인터록 빡세게.

**Phase 4 — "실전 파일럿"**
- 실제 소방서와 통제된 훈련장 파일럿. 규제·인증·신뢰성.

> 솔직 코멘트: Phase 0~1이 진짜 데모 임팩트 다 가져간다. 거기부터 만들자.
