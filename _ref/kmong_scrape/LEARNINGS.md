# 크몽 에이전시 포트폴리오 탐방 → FRIZE 랜딩 본문 개선 레퍼런스

> 목적: FRIZE 랜딩의 **히어로 밑 본문**이 "AI가 뽑은 SaaS 템플릿"처럼 보이는 문제를 고치기 위해,
> 크몽에서 **실제 에이전시가 만든 웹/랜딩 포트폴리오** 25개 + 라이브 사이트 3개를 캡쳐해 분석.

## 0. 어떻게 수집했나 (재현용)
- Playwright(chromium) + `ignoreHTTPSErrors:true` (이 환경 프록시가 TLS 가로채서 인증서 에러남 → 필수)
- 크몽 포트폴리오 검색: `https://kmong.com/search?type=portfolios&sort=RANKING_POINTS&service=web&keyword=홈페이지 제작`
  - 결과 href에 `portfolio_id=XXXX` 가 박혀있음 → `/portfolio/view/{id}` 로 매핑
  - 랭킹순 web 필터로 페이지 1~4 긁어 120개 후보 확보 (`web_portfolios.json`)
- 각 뷰 페이지: **끝까지 슬로우 스크롤**(900px씩, 200ms 대기)로 lazy 이미지/더보기 펼침 트리거 후 `fullPage` 캡쳐
- 스크립트: `collect2.mjs`(수집) / `capture.mjs`(포폴 캡쳐) / `live.mjs`(라이브 사이트 캡쳐)
- 산출물: `shots/pf_*.png`(25), `shots/live_*.png`(3), 메타 `results.json`

## 1. 한 줄 결론
크몽 본문 "포트폴리오"는 크몽 페이지가 아니라 **에이전시가 만든 실제 사이트의 캡쳐 스택**이다.
프로들이 본문에서 쓰는 무기는 화려한 위젯이 아니라 **(1) 리듬 (2) 풀블리드 비주얼 밴드 (3) 강한 타입+모노 캡션 (4) 절제된 컬러**다. FRIZE 히어로는 이미 이걸 갖췄고, 본문만 못 따라가고 있음.

## 3. 프로 본문이 공통으로 하는 것 (= FRIZE에 적용할 규칙)

### A. 섹션 리듬을 "배경"으로 만든다 (가장 중요)
- 프로 사이트는 섹션마다 **배경색/명도를 통째로 바꾼다**: 라이트 → 다크 → 풀블리드 사진 → 라이트.
  스크롤하면 화면 전체 색이 쿵쿵 바뀌면서 "구간"이 생김.
- FRIZE 현재: 거의 다 같은 톤 위에 카드만 얹혀서 **밋밋한 한 덩어리**로 읽힘.
- **적용:** pillars=라이트, VISOR=다크 풀블리드, SCOUT=라이트, cinema=사진 풀블리드, nums=잉크블랙… 이런 식으로 명암 교차를 의도적으로.

### B. 풀블리드 비주얼 밴드 (가로 100%, 텍스트는 위에 얹기)
- rlwrld/HEADIT 모두 **화면 꽉 채운 영상/사진 위에 모노 캡션 한 줄 + 큰 헤드라인**.
- FRIZE의 `cinema` 섹션이 이미 이 형태 → **개수를 늘리고** hw 섹션 사이사이에 더 박아 리듬 줄 것.

### C. 타이포 대비 극단화
- 헤드라인은 **더 크고 더 굵게**(거의 화면 절반 폭), 본문은 **작고 회색**, 그 위에 **모노스페이스 대문자 라벨**(`SITUATIONAL AWARENESS` 같은 거 — FRIZE 이미 함, 더 밀어붙일 것).
- 한국 에이전시 특: **컬러 키워드 강조**("MZ세대의 <빨강>감성</빨강>" 처럼 핵심 단어만 포인트색). FRIZE도 `<em>` 강조 있음 → 색 대비 더 키우기.

### D. 컬러는 1개로 절제
- 잘 만든 곳은 강조색 **딱 하나**(teal/보라/빨강)를 전 섹션에 일관 적용. FRIZE는 소방=레드 톤이 자산 → **레드 하나만** 악센트로 끝까지.

### E. 숫자/스펙 섹션은 살아있다
- nums 같은 "by the numbers"는 프로도 자주 씀. 단, **큰 숫자 + 얇은 라벨 + 구분선**으로 더 시원하게.

### F. 디바이스/제품 목업
- HEADIT는 폰 목업에 화면 얹음. FRIZE는 콕핏/트윈 UI 스샷을 **노트북/모니터 목업**에 얹으면 즉시 "실제 제품" 느낌.

### G. 인터랙션은 "리빌"이면 충분
- 화려한 스크롤재킹(HEADIT)은 캡쳐도 안 될 만큼 과함. 프로 대부분은 **스크롤 시 페이드/슬라이드 업 리빌**(FRIZE `data-reveal` 이미 있음)로 충분. 타이밍/거리만 다듬으면 됨.

## 4. FRIZE 본문 구체 개선 액션 (우선순위)
1. **섹션 배경 교차** 도입 — pillars/hw/cinema/nums에 라이트·다크·풀블리드 번갈아 (규칙 A). ← 효과 최대
2. **cinema 풀블리드 밴드 1~2개 추가**, hw 섹션 사이에 배치해 리듬 (규칙 B)
3. **헤드라인 스케일 업 + 모노 라벨/레드 강조 강화** (규칙 C/D)
4. **콕핏/트윈 UI를 디바이스 목업**에 얹은 섹션 신설 (규칙 F)
5. nums 타이포 정리, reveal 타이밍 튠 (규칙 E/G)
